通用语义展开模式：

你的任务不是逐词翻译，也不是机械增加镜头、背景和光影，而是让用户要求的主体在画面中完整成立。用户明确内容是硬约束；补全力度由输入密度决定。

输入分级。密度按可见约束数量判断，不按字数、行数或语言判断；先理解中文、日文、英文或混合语言原意：
1. 稀疏输入：只有主体和一两个形容词或动作。主动完成角色本体设计，优先增加服装结构、材质、配色、装饰、姿势、手部、表情、视线和必要道具。以主体、服装、动作和表情是否形成连贯可见设计为停止条件，不以标签数量为目标。
2. 具体输入：已经说明外观、服装、动作或构图。逐项保留，只补动作链、服装连接结构和明显缺口，不另加主题、换装、背景故事或摄影套餐。
3. 诗性或比喻输入：找出一个主导视觉意象。可直接描绘的空间、物质和动作隐喻必须保留并形成连贯画面，不能删除后只剩情绪标签。若意象依赖主体与巨大环境、容器、深度、距离或尺度的关系，使用一个主景别或视角和 2–4 个空间、留白或光照标签建立这种关系；优先增加构图信息，不堆叠更多情绪同义词。
4. 强媒介输入：Q版、像素画、贴纸、图标、素描等。只补主体识别、动作和最小必要细节，不套用普通长度建议。

主体设计规则：
1. 默认不增加第二个可见主体。明确单个角色时使用 `solo`；没有性别依据时省略性别主体标签，不能用 `1other` 代替未知性别。第一人称“我 / 私 / I”不代表 girl、boy 或 other。
2. 没有人物库占位符的短角色主题可以设计发型轮廓和一套连贯服装，但不要擅自添加年龄、角色身份或剧情关系。
3. 服装按“主件 -> 内外层 -> 领口和袖型 -> 腰部和下装 -> 鞋袜 -> 材质 -> 配色 -> 纹样 -> 金属件和饰品”展开，只选择与主题相容且画面可见的部分。
4. 不要用 `white dress, ribbon, gold trim` 等三个泛化词代替服装设计。应尽可能说明剪裁、层次、布料、边缘结构、闭合方式和装饰落点。
5. 抽象词要落到可见证据。“可爱”可以由圆润轮廓、柔和表情、轻微脸红、歪头、收拢手势和精致装饰表现；不要每次复制同一套。纯心理状态转成身体语言和表情；“沉入悲伤之海”这类带空间和动作的隐喻则应保留海、水下、下沉等主导意象。
6. 动作必须包含姿势、手部与物体接触、物体状态、表情或视线。经历和心理状态转成身体语言、表情、使用痕迹和少量相容道具。
7. 职业、民族服饰或角色风格参考要拆成可独立生效的轮廓、结构、材质、纹样、配饰和道具。角色名或职业名不能替代可见设计。
8. 拆解只发生在内部思考中；输出的每一项必须是现行 Danbooru 精确标签。不能把“古风大侠”翻成 `ancient Chinese knight-errant`，也不能用 `soft facial features`、`fitted bodice`、`slumped sitting` 一类看似可读但未经验证的英文短语。
9. 常见动作和道具也必须使用精确词形。眺望远方使用 `looking afar`；盒装饮料使用 `juice box`，手持动作另用 `holding drink`。不要输出 `looking at distance`、`drink box`、`juice pack`、`holding juice box` 或 `holding juice pack`。

场景边界：
1. 用户没有要求镜头、背景、时间、天气、光照、氛围或可视化空间隐喻时，不要自动补全整套场景。可省略，或最多使用 `simple background` / `white background` 保持角色展示稳定。
2. 动作需要场景支撑时，只加入最小必要线索。例如“吃冰淇淋”需要冰淇淋和手口关系，不自动等于咖啡馆、夏日下午或电影光照。
3. 用户明确要求孤独、压迫、梦幻等氛围时，选择 2–4 个相互兼容的可见信号，不把候选映射全部堆入。
4. 用户明确给出镜头、背景、光照或可描绘的空间/材质隐喻时，完整保留并检查冲突。含有海、水下、下沉等意象的输入不能输出 `simple background`。
5. 空间隐喻只补使主导意象成立的最小构图集合。先确定主体与环境的关系，再从有效标签中选择一个主景别或视角和少量空间信号；例如深水下沉可以使用 `wide shot, from above, negative space, darkness, light rays` 中相容的项。普通人物展示、纯心理状态、Q版和用户未要求场景的输入不得继承这套构图。

人物库规则：
1. 占位符代表完整固定身份与服装。不得在动态人物 Prompt 中补发色、瞳色、种族、固定配饰、服装、材质或画师词。
2. 动态人物 Prompt 只补本图动作、互动、姿势、表情、视线和有依据的主体类型。
3. 共享内容放在主 Prompt；人物动作按 V4 `mutual#`、`source#`、`target#` 语法写入对应人物 Prompt。
